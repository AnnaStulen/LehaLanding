Блок должен показать, что курс построен не вокруг абстрактных лекций, а вокруг специализированной рабочей инфраструктуры:

- модули исполнения;

- позиция;
- журнал/разбор сделок;

Главная мысль:

**Студент учится работать в среде торгового деска, а не просто смотреть уроки.**

## Композиция

Не делай стандартный шаблон “текст слева, картинка справа”.

Нужна другая структура:

1. Сначала широкий software visual на всю ширину лендинга.
2. Под ним или поверх него — живой слой смыслов.
3. Текст не должен выглядеть как Google Docs или обычный список.
4. Используй разные по размеру и весу смысловые фрагменты:
   - крупные короткие тезисы;
   - маленькие technical labels;
   - floating tags;
   - callouts;
   - status chips;
   - mini-panels;
   - разные размеры текста, но в единой системе.

Композиция должна ощущаться как “software command surface” или “trading desk console”:

- широкий экран занимает главную роль;
- вокруг него находятся подписи к зонам;
- часть текста может быть поверх темных участков visual-frame;
- часть текста может быть под ним как неровная, но управляемая сетка;
- блок должен быть хаотичным по ритму, но аккуратным по дизайну.

## Как использовать `program screen 1.png`

Используй картинку из:

`assets/program-screen-1.png`

Используй ее как главный широкий visual на всю ширину блока.

Обработка:

- НЕ затемнять глобально;
- НЕ добавлять black overlay на всю картинку;
- сохранить картинку близкой к оригиналу: интерфейс должен быть читаемым, ярким и доказательным;
- если поверх картинки есть подписи, использовать только локальные темные подложки под текст;
- привести к общему blue/cyan стилю через frame, glow, тонкие линии и callouts, а не через затемнение всего изображения;
- сделать square technical frame;
- добавить glow по краям;
- добавить легкий grain/scanline, если это выглядит дорого;
- не делать ее плоской вставленной картинкой.

Важно: первый приоритет для `program screen 1.png` — сохранить оригинальную читаемость. Не ставь `opacity` ниже 0.85, не используй `brightness()` ниже 0.95 и не перекрывай скрин сплошным черным слоем.

Подписи/callouts адаптировать под видимые зоны этого экрана:

- `Control panel` — панель конфигураций, счетчики и состояния алгоритмов;
- `Order flow` — стаканы, flow-ленты и лимитная ликвидность;
- `Liquidity map` — тепловые зоны, FlowScape и плотность ликвидности;
- `Chart review` — нижние графики, реакция цены, контекст позиции и качество решений.

## Интерактив

Блок не должен быть статичным.

Нужно добавить:

- hover на широкий visual: мягкий zoom-in, glow, небольшая глубина;
- hover на зоны/callouts: подсветка соответствующей области экрана;
- hover на mini-panels/tags: border-light, glow, slight translate;
- можно добавить moving scanline или медленное движение data-line поверх visual;
- можно добавить cursor-follow light внутри software frame, если это не перегружает;
- клик по главному visual не нужен: не добавлять lightbox/увеличение для `assets/program-screen-1.png`.

Все motion:

- через transform, opacity, filter, CSS variables, SVG stroke;
- без layout shift;
- с `prefers-reduced-motion`;
- без дешевого мигания и rainbow.

## Текстовые элементы

Не писать длинные абзацы.

Лучше использовать короткие смысловые состояния:

- `Execution terminal`
- `Risk layer`

- `Market data`
- `Trade review`

- “Каждый модуль курса привязан к действию: наблюдать, исполнять, контролировать, разбирать.”

Можно использовать микс русского и технического английского, потому что это fintech/trading интерфейс.
